# Java'da Paketler, Import Kullanımı ve Proje Düzeni

## H1: Java Paket Sistemi, Import Mekanizması ve Modüler Proje Yapılandırması

### H2: Java Paket Sisteminin Temelleri ve Adlandırma Kuralları
- **Kavram**: Paket nedir? Java'da sınıfların organize edilme biçimi, ters domain adı kuralı (com.example.proje)
- **Örnek**: Standart Java paketleri (java.util, java.io) ve özel paket yapıları
- **Uygulama**: Kendi paket hiyerarşimizi oluşturma (com.egitim.uygulama.model, com.egitim.uygulama.service)
- **Değerlendirme**: Paket adlandırma hatalarını tespit etme alıştırması

### H2: Paket Bildirimi ve Dizin Yapısı ile Çalışma
- **Kavram**: `package` anahtar kelimesi, fiziksel dizin yapısı ile paket yapısının eşleşmesi
- **Örnek**: Çok katmanlı bir projede dizin organizasyonu gösterimi
- **Uygulama**: **Kod Örneği** - Üç farklı pakette sınıf oluşturma ve derleme
  ```java
  // com/egitim/arac/Kasa.java
  package com.egitim.arac;
  public class Kasa { ... }
  
  // com/egitim/motor/Motor.java
  package com.egitim.motor;
  public class Motor { ... }
  ```
- **Değerlendirme**: Yanlış dizin yapısıyla derleme hatası oluşturma ve çözümleme

### H2: Import Mekanizması ve Kullanım Şekilleri
- **Kavram**: `import` deyimi, tam nitelikli isim (fully qualified name) kullanımı, wildcard import (`*`)
- **Örnek**: Aynı isimli sınıfların farklı paketlerden import edilmesi durumu
- **Uygulama**: **Kod Örneği** - Üç farklı import türünü karşılaştırmalı gösterme
  ```java
  // Tam nitelikli isim kullanımı
  java.util.Date bugun = new java.util.Date();
  
  // Tek sınıf import
  import java.util.ArrayList;
  
  // Wildcard import
  import java.util.*;
  ```
- **Değerlendirme**: Çakışan import durumlarını çözme alıştırması (java.util.Date vs java.sql.Date)

### H2: Static Import ve Özel Kullanım Durumları
- **Kavram**: `import static` ile sabit ve statik metotların doğrudan kullanımı
- **Örnek**: Math sınıfı sabitlerinin (PI, E) ve metotlarının (sqrt, abs) static import ile kullanımı
- **Uygulama**: **Kod Örneği** - Static import öncesi ve sonrası kod karşılaştırması
  ```java
  // Static import olmadan
  double alan = Math.PI * Math.pow(yaricap, 2);
  
  // Static import ile
  import static java.lang.Math.*;
  double alan = PI * pow(yaricap, 2);
  ```
- **Değerlendirme**: Static import aşırı kullanımının okunabilirlik etkisi üzerine tartışma

### H2: Modüler Proje Yapılandırması ve Best Practices
- **Kavram**: Maven/Gradle proje yapısı, kaynak kod ve test dizinleri ayrımı, module-info.java (Java 9+)
- **Örnek**: Gerçek dünya proje yapısı şeması (src/main/java, src/test/java, src/main/resources)
- **Uygulama**: **Kod Örneği** - module-info.java ile modül tanımlama
  ```java
  // module-info.java
  module com.egitim.uygulama {
      exports com.egitim.uygulama.api;
      requires java.sql;
      requires transitive java.logging;
  }
  ```
- **Değerlendirme**: Proje yapısı hatalarını bulma ve düzeltme alıştırması (eksik dizin, yanlış paket yapısı)

### H2: Paket ve Import Yönetiminde İleri Düzey Konular
- **Kavram**: Paket görünürlüğü (public, protected, default), JAR dosyaları ve classpath yönetimi
- **Örnek**: Farklı JAR'lar arasında sınıf çözümleme ve sıralama
- **Uygulama**: **Kod Örneği** - Kendi JAR'ımızı oluşturma ve başka projede kullanma
  ```bash
  # JAR oluşturma
  jar cf arac.jar com/egitim/arac/*.class
  
  # JAR'ı classpath'e ekleme
  javac -cp arac.jar Main.java
  java -cp arac.jar:. Main
  ```
- **Değerlendirme**: Classpath problemlerini giderme senaryosu (NoClassDefFoundError çözümü)

### H2: Proje Alıştırması - Çok Katmanlı Mini Proje
- **Kavram**: Öğrenilen tüm kavramların entegrasyonu
- **Örnek**: Kütüphane yönetim sistemi için paket yapısı tasarımı
- **Uygulama**: **Kod Örneği** - Tam bir proje yapısı oluşturma
  ```
  com/
  └── kutuphane/
      ├── model/
      │   ├── Kitap.java
      │   └── Uye.java
      ├── service/
      │   ├── KitapService.java
      │   └── OduncService.java
      ├── util/
      │   └── TarihUtils.java
      └── Main.java
  ```
- **Değerlendirme**: Projenin paket yapısını, import kullanımını ve modülerliğini değerlendirme rubriği
